From prompt to prod

21 April 2026 · CroAI · Vedran Mandić — backend engineer at Similarweb

Takeaways (short version)

  • Spend real time on context: globs for rules, lean skills, fresh threads for unrelated work.
  • Treat plan and review as first-class steps; execution is cheaper when the plan is right.
  • Prefer scoped automation (skills, MCPs, scripts you own) over giant always-on rule packs.
  • Verify everything that touches production paths, credentials, or customer data.

How to self-audit yourself

On the self-audit page, use the prompt builder (click choices, type only your path), then paste the generated starter and the full rubric into your preferred coding agent (Cursor, Claude Code, Copilot in the editor, or similar) that can read your files on disk. The rubric expects bounded scans of real paths; a plain chat with no filesystem access will not work. The same page has the full A–K guide. Offline templates for copy-paste are in the repo README.md.
